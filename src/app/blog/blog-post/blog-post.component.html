Boilerplate HTML Generator (Bash Script)

COMM429 Submission

January 25, 2024

Today, we're jumping into the shallow end of the scripting pool. No fancy jargon—just pure, easy, command-line fun.

One of the most tedious parts of starting a new website has got to be creating the file structure to house all your HTML, CSS and JS. Well, imagine this: you're staring at your screen, faced with a blank project folder. You know what you want to build, and instead of having to spend the 5 minutes to create the file scaffolding, you click one button and BAM.

That's where scripting comes in. With just a few clever lines of code, you can automate the boring stuff, scaffold out your project structure, and get straight to the fun part—coding. In this post, we're going to walk you through the process of creating a bash script that'll make your life easier and your projects more efficient. Let's Get Started!

NOTE: This guide is for Mac users using bash commands but links to windows related articles are provided below!

We can accomplish this in three easy steps:

  • Function Use Validation
  • Directory Check and Creation
  • Creating Files and Adding Content

The first part of this involves creating a simple .sh file that will be our script file that we run to generate our project's folder contents. We will name ours webScaffold.sh and we can create in any folder or directory and we will move it at the end. Then open the .sh file with your favorite text editor and follow below:

Function Use Validation

1
#!/bin/bash
2
if [ -z "$1" ]; then
3
echo "Usage: $0 <directory_name>"
4
exit 1
5
fi
6
directory_name="$1"

In this section, the script checks if there's at least one argument provided when executing the script. If not, it prints a message indicating the correct usage of the script and exits with a status of 1. The first argument ($1) is then assigned to the variable directory_name.

Directory Check and Creation

7
if [ -d "$directory_name" ]; then
8
echo "Directory '$directory_name' already exists."
9
exit 1
10
mkdir "$directory_name"
11
echo "Directory '$directory_name' created successfully."

Here, the script checks if a directory with the name stored in $directory_name already exists. If it does, it prints a message indicating that the directory already exists and exits with a status of 1. If the directory doesn't exist, it creates one using the mkdir command with the name stored in $directory_name, and then prints a success message.

Creating Files and Adding Content

12
cd "$directory_name"
13
mkdir images scripts styles
14
touch index.html
15
touch styles/style.css
16
echo "<!DOCTYPE html>" >> index.html
17
echo "<html>" >> index.html
18
echo " <head" >> index.html
19
echo " <meta charset=\"utf-8\">" >> index.html
20
echo " <meta name=\"viewport\" content=\"width=device-width, initial-scale=1.0\">" >> index.html
21
echo " <link rel=\"stylesheet\" href=\"styles/style.css\">" >> index.html
22
echo " </head>" >> index.html
23
echo " <body>" >> index.html
24
echo " </body>" >> index.html
25
echo "</html>" >> index.html

In this final section, the script navigates (cd) into the directory specified by $directory_name. Then, it creates three subdirectories - images, scripts, and styles. After that, it creates two files - index.html and styles/style.css using the touch command. Following this, it appends HTML content to index.html file using echo commands, effectively creating a basic HTML structure with an empty body and a link to the CSS file.

Wrap Up and Execution

After saving the shell script (.sh), it's recommended to move it to the bin folder for easy access and execution. The bin folder could be located in the current user's profile or in the system's bin directory. Make sure you have the ability to see hidden files and simply navigate to the terminal of your choice, retrieve the file path to your .sh script, and type the following:

Mac-mini ~ %
mv ~/Desktop/MyFile.sh ~/bin/

The above will move our script to a folder that is accessable everywhere so now we can go to any directory we want to create a new projects and run the following line:

Mac-mini ~ %
./webScaffold myProject

After execution you will see all the folders and files created within the directory we chose!
